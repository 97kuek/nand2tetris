# Nand to Tetris (コンピュータシステムの理論と実装)

このリポジトリは、書籍『コンピュータシステムの理論と実装 ―モダンなコンピュータの作り方』の学習用ワークスペース。論理ゲート（NAND）から始まり、最終的にテトリスのようなゲームが動くコンピュータシステム全体をボトムアップで構築していく。

## ディレクトリ構成

* `projects/`: 各章ごとのプロジェクトディレクトリ。実装のベースとなるひな形ファイルや、テスト用のスクリプト（`.tst`）、期待される出力結果（`.cmp`）が含まれる。
  * `01/` 〜 `05/`: ハードウェアの構築（論理ゲート、ALU、メモリ、CPUなど）
  * `06/`: アセンブラの作成
  * `07/` 〜 `08/`: バーチャルマシン（VM）の作成
  * `09/`: Jack言語でのプログラミング
  * `10/` 〜 `11/`: コンパイラの作成
  * `12/`: オペレーティングシステム（OS）の実装
* `tools/`: プロジェクトを進めるために必要なソフトウェアツール群。著者から提供されているシミュレータやテストツールなどが格納されている。

## ツールの使い方

プロジェクトを進める際は、基本的に `tools/` フォルダ内にあるバッチファイル（`.bat`）をダブルクリックするか、コマンドプロンプトやPowerShellから実行してGUIツールを起動する。

主なツール:
* **HardwareSimulator.bat**: ハードウェア構築（プロジェクト1〜3、5）で使用するシミュレータ。`.hdl`（ハードウェア記述言語）で書いた回路をテストする。
* **Assembler.bat**: アセンブリ言語を機械語に変換するアセンブラ（プロジェクト6以降のテスト等で使用）。
* **CPUEmulator.bat**: CPUの動作をシミュレートするエミュレータ。機械語プログラムの実行確認に使用する。
* **VMEmulator.bat**: バーチャルマシンのシミュレータ（プロジェクト7〜8で使用）。
* **JackCompiler.bat**: Jack言語用のコンパイラ（プロジェクト10〜11で使用）。

### 基本的な開発の流れ（Hardware Simulator の場合）

1. **実装を行う**
   * テキストエディタを開き、`projects/01/` などのディレクトリにある目的の `.hdl` ファイル（例：`Xor.hdl`）に論理回路（`PARTS:` の下部）を記述して保存する。
   
2. **シミュレータを起動する**
   * `tools/HardwareSimulator.bat` をダブルクリックして起動する。

3. **HDLファイルを読み込む**
   * GUI左上のメニューから **File** ＞ **Load Chip** をクリック。
   * 先ほど編集した `.hdl` ファイル（例: `projects/01/Xor.hdl`）を選択して開く。
   * （※画面中央の HDL 領域に記述したコードが、左側にピン情報が表示されるはず）

4. **テストスクリプトを読み込む**
   * GUI左上のメニューから **File** ＞ **Load Script** をクリック。
   * テスト対象と同じ名前の `.tst` ファイル（例: `projects/01/Xor.tst`）を選択して開く。

5. **テストを実行する**
   * 画面上部にある **再生ボタン（単一の右向き矢印 `►`）** または **早送りボタン（二重の右向き矢印 `►►`）** をクリックして実行する。
     * `►`：1ステップずつ実行（入出力ピンの値がどう変わるか追うのに便利）
     * `►►`：最後まで一気に実行
   * 画面下部のステータスバーに **「End of script - Comparison ended successfully」** と表示されればテストクリア！
   * もし「Comparison failure at line ○○」などと出た場合は実装が間違っている。HDLファイルを修正・保存し、再度 **File ＞ Load Chip** からやり直す。

## 前提条件・注意事項

* 付属ツールは **Java** で作られているため、PCにJava実行環境（JRE/JDK）がインストールされている必要がある。ツールが起動しない場合は、Javaがインストールされているか、パスが正しく通っているか確認する。
* ファイルやディレクトリのパスに日本語（全角文字）や空白が含まれていると、ツールが正常に動かない場合がある。
